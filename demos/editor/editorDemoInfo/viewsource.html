No matter what, there are always some people who want to work with the source view of a document in addition to editing it as a WSYWIG RichText document.  This is common in blog environments and forums, as well as many intranet applications.   Wouldn't it be great if those applications could use the dijit.Editor as well. So as of the Dojo Toolkit, version 1.4, the editor has a View Source plugin.

Once the view source plugin is enabled, the user can click the view source icon (Or use the key command CTRL-SHIFT-F12) to toggle the view into source mode.  Clicking it again (or using the key command again) returns the editor to WYSIWYG mode.   With the View Source plugin, you application has the best of both words; a source view for users who are technical and like working in source, and a WYSIWYG view for those more familiar with 'word processor' style editing.

Try it!

Click the view source icon in the editor on the right.   You will see the view toggle to source more.  Also notice all the Rich Text editing options in the toolbar are automatically disabled.  Clicking the icon again will restore Rich Text editing mode.
